High-Priority Action Plan (Mandatory Before Bihar Release)

Conversation Reset Mechanism and User Sessions

Users can reset anytime after submitting the problem statement and the solution. Restrict session flow to 5 times/user. Modify the last message from a "Thank You" to also prompt the user ("Let me know if you have a better idea!") where the user profile remains the same and clicking on Reset wipes off the user profile data.

Dynamic Mandatory Field for Company Name

Remove the field regarding company registration status and Company name entirely from the flow. This information is not as crucial as other data points.

Missing Essential Field (Website Name for MVP Stage)

Add 'Website Name' as a distinct field in the `Data Fields to Capture` schema, ensuring it's included for `MVP Stage` users. It should be 'User Compulsion: true' for MVP stage to gather critical information.

Multi-Link Display Issue in Pitch Document

Review and adjust the formatting logic to correctly handle and display multiple links. This may involve using bullet points, numbered lists, or clear line breaks to ensure readability and prevent truncation.

Inconsistent Stage-Specific Elaboration (e.g., Solution)

Review the `Description` and guidance provided for key fields (like 'Solution') in `Data Fields to Capture`. Introduce more detailed, stage-specific elaboration instructions for pitch document creation to ensure consistency.

Hindi Language Support Deficiencies

Prioritize a deep dive into the bot's NLP capabilities for Hindi. This requires integrating a more robust Hindi NLP model, extensive testing with various Hindi inputs (including half-sounds and matras), and explicit instruction for accurate Devanagari script rendering.

Flexible Data Input (Age vs. DOB)

Remove the separate question for age/DOB. Instead, ask the user to mention their age within their general introduction, allowing for more natural input.

Limited Bot Assistance (Editing Phase / No In-Line Editing)

Expand bot functionality for the "make edits" phase, allowing users to specify which field they want to edit *during* the gathering for that field. This enables re-engagement in a focused information-gathering loop for specific fields.

Important Improvements (To look out for in the future)

Restrictive Response Formats

Review all current "Yes/No" forced questions. Modify rules to interpret more natural language for confirmations or binary choices, reducing explicit "Yes/No" prompts.

Stale WhatsApp Button Handling

Develop robust logic to handle out-of-context or stale button presses. Bot should acknowledge confusion and redirect to current conversation context or offer clear navigation.

Missing Pitch Document Data (City & State)

Ensure 'Location' field (or new separate fields for city/state) is explicitly collected and included in the "About the Founding Team" or a dedicated "Contact/Location" section within the Pitch Document Creator's output.

Missing Feature (Share Location)

Investigate the feasibility of integrating a "Share Location" feature via WhatsApp's API for streamlined location data collection, enhancing user convenience.

Bihar Release: Lean Question Flow

This streamlined flow prioritizes essential information for quicker initial pitch document generation, with options for deeper elaboration based on user choice.

1. Welcome & Language Selection (English/Hindi)
↓
2. Core Idea & Problem/Solution Articulation
↓
3. Founder Introduction (with embedded Full Name, Age, Email, City & State)
↓
4. User Choice: Elaborate on Key Elements (Revenue, GTM, etc.) OR Generate Initial Pitch
↓
5. Final Call: Any external links or additional information for the pitch document?
↓
6. Display Pitch Summary & Offer Final Refinement OR Generate Full Pitch Document

Impact of the Above Process:

  • Reduces the number of mandatory questions from 16 to a lean 7.
  • Offers flexibility to dive into deeper details/fields for interested users.
  • Brings in a more conversational flow to enhance user experience and reduce probing.
